🛠️ Service: Webhook ServiceLayanan ini bertanggung jawab untuk meneruskan data pesan (payload) yang telah diproses oleh WhatsApp Gateway ke aplikasi Laravel melalui protokol HTTP.🎯 Fungsi UtamaIntegrasi Data: Menghubungkan aktivitas WhatsApp (pesan masuk, status koneksi) ke database atau logika bisnis di Laravel.Keamanan Gateway: Memastikan setiap data yang dikirim memiliki identitas melalui Header khusus.Monitoring: Memberikan log mendetail di terminal untuk memantau apakah Laravel menerima data dengan sukses atau menolaknya.🔐 Keamanan & HeaderUntuk mencegah akses ilegal langsung ke endpoint Laravel Anda, service ini menyertakan proteksi berikut:x-api-key: Kunci rahasia yang diambil dari process.env.API_KEY. Laravel wajib memvalidasi header ini sebelum memproses data.Content-Type: Diatur secara eksplisit ke application/json untuk memastikan kompatibilitas parsing di sisi server penerima.📡 Detail Method: sendToLaravel(payload)Method ini bersifat asynchronous dan dirancang untuk ketahanan sistem tinggi.PropertiDetailInput Parameterpayload (Object): Data lengkap pesan, sender, media_id, dll.Timeout10 Detik (Mencegah Memory Leak jika Laravel lambat merespon).Retry Logic(Opsional) Disarankan dipanggil di dalam sistem antrean (Queue).Alur Kerja:Validasi Konfigurasi: Mengecek apakah URL tujuan sudah diatur di .env.Logging Payload: Mencetak JSON lengkap ke konsol untuk keperluan debugging developer.HTTP Request: Melakukan POST request menggunakan library axios.Error Handling:Response Error: Menampilkan pesan error spesifik dari Laravel (misal: error 422 validasi atau 500 server error).Request Error: Mendeteksi jika server Laravel mati atau URL tidak dapat dijangkau.⚠️ Penanganan Error (Debugging)Service ini memberikan pesan error yang informatif di terminal Node.js:❌ Laravel Error (401): API Key Anda tidak valid di sisi Laravel.❌ Laravel Error (404): URL Webhook salah atau route di Laravel belum dibuat.❌ Tidak ada respon dari Laravel: Masalah jaringan atau server Laravel sedang down.
